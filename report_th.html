01 — ภาพรวม

ทำไมต้องมีระบบนี้?

จากการตรวจ listing 20 รายการแรกบน PropertyScout.co.th พบปัญหาเชิงระบบหลายอย่างที่ส่งผลต่อทั้งอันดับ SEO และความน่าเชื่อถือของประกาศ

⚠️

ปัญหาที่พบ (จากตัวอย่าง 20 listing)

• 20/20 ไม่มีค่าส่วนกลาง (CAM) ค่าน้ำ ค่าไฟ
• 13/20 ไม่มีชื่อนักพัฒนาโครงการ (ลด trust signal)
• 15/20 หัวข้อ/ชื่อประกาศยังไม่ใช้ keyword ที่คนค้นหาจริง
• 8/20 ระยะทาง BTS/MRT หรือชื่อสถานีไม่ถูกต้อง
• 0/20 มี FAQ schema สำหรับ AI search

🎯

เป้าหมายของระบบ

• วิเคราะห์ listing ทุกตัวอัตโนมัติ ผ่าน AI 11 agent
• เติมข้อมูลที่ขาด + ตรวจสอบความถูกต้อง
• เขียน title, headline, description, FAQ ใหม่ — ภาษาไทยและอังกฤษ
• ให้คะแนนคุณภาพ (PASS / BORDERLINE / FAIL) พร้อม priority action
• ส่งออกเป็น JSON + CSV ให้ทีม content นำไปใช้ต่อได้ทันที

02 — ระบบทำงานอย่างไร

Pipeline ของ AI Agent ทั้ง 11 ตัว

แต่ละ listing จะถูกส่งผ่าน agent ทั้ง 11 ตัวตามลำดับ เปรียบเสมือนทีมงานเสมือนจริง (Virtual Office) ที่แต่ละคนรับผิดชอบงานของตัวเอง แล้วส่งต่อให้คนถัดไป

1
KANYA INTAKE
ดึงข้อมูลจากหน้า listing — ชื่อ ประเภท ราคา พื้นที่ ชั้น ห้องนอน/น้ำ จำนวนรูป วิดีโอ และตรวจหา listing ที่อาจซ้ำกัน
2
PANSA ENRICH
ค้นข้อมูลที่ขาด — ชื่อนักพัฒนา ปีสร้าง จำนวนยูนิต ระยะทางถึง BTS/MRT ห้าง โรงพยาบาล โรงเรียน และสิ่งอำนวยความสะดวกของโครงการ
3
SUWAN AUDIT
ประเมินความสมบูรณ์ของข้อมูลเบื้องต้น และ flag listing ที่มีรูปน้อย / ไม่มีค่า CAM / ไม่มีชื่อนักพัฒนา
4
PRIYA RESEARCH
เปรียบเทียบราคาต่อ ตร.ม. กับตลาด ดูคู่แข่งในย่านเดียวกัน และค่า CAM ที่เหมาะสมสำหรับโครงการประเภทนี้
5
NIRAN FACTCHECK
ตรวจสอบความสมเหตุสมผลของระยะทาง BTS/MRT ชื่อสถานี ปีสร้าง จำนวนชั้น และราคาต่อ ตร.ม. ที่ผิดปกติ
6
TAWAN SEO
ปรับ title และ keyword ให้ติดอันดับ Google ทั้งภาษาไทยและอังกฤษ — ใช้คำที่คนค้นหาจริง เช่น "คอนโดให้เช่าใกล้ BTS"
7
WICHAI COPYWRITER
เขียน headline, description, meta description ใหม่ทั้ง TH/EN โดยอิงข้อเท็จจริง ไม่ใช้คำคลุมเครือ เช่น "luxury" โดยไม่มีหลักฐาน
8
DAOW AEO
สร้าง schema markup และชุด FAQ (TH/EN) เพื่อให้ ChatGPT, Perplexity, Gemini ดึงข้อมูลไปตอบคำถามผู้ใช้ได้ง่ายขึ้น
9
PLOYCHIT OPTIMISE
รวมผลจาก agent 6-8 ให้ keyword TH/EN ตรงกัน, FAQ ครบทุกภาษา และเรียงลำดับ issue ที่สำคัญที่สุดก่อน
10
PHITCHAYA SCORE
คำนวณคะแนนสุดท้าย จาก completeness × 0.4 + accuracy × 0.3 + appeal × 0.3 และให้เกรด A–D
11
FIAT PUBLISH
ตัดสินใจ — PASS เผยแพร่ได้ทันที / BORDERLINE ต้องแก้ priority issue ก่อน / FAIL ส่งกลับไปเริ่มใหม่

TH: "คอนโด 1 นอน ห่าง BTS เพลินจิต เพียง 120 เมตร ใจกลางเมือง"

BTS เพลินจิต อยู่ห่างจากตึกแค่ไหน? / How far is BTS Phloen Chit?
BTS เพลินจิตอยู่ห่างเพียง 120 เมตร เดินประมาณ 2 นาที เชื่อมต่อสยาม อโศก และทุกสายของ BTS ได้สะดวก
ชาวต่างชาติเช่าหรือซื้อ Noble Ploenchit ได้ไหม? / Can foreigners rent or buy here?
ได้ โครงการเป็นฟรีโฮลด์และมีโควตาต่างชาติ เป็นที่นิยมในกลุ่มผู้เช่าและผู้ซื้อต่างชาติ
โครงการมีสิ่งอำนวยความสะดวกอะไรบ้าง? / What facilities are available?
สระว่ายน้ำ ฟิตเนส กอล์ฟซิมูเลเตอร์ พื้นที่ทำงานร่วม สวนและบาร์บีคิว ที่จอดรถในร่ม รักษาความปลอดภัย 24 ชม.

ระบบยังเพิ่ม schema markup (Apartment), market comparison (902 บาท/ตร.ม. = ราคากลางของย่าน), และ investment notes สำหรับนักลงทุนต่างชาติ — ทั้งหมดนี้ออกมาในไม่กี่นาทีต่อ listing

06 — ผลกระทบในอนาคต

ถ้านำไปใช้งานจริง จะเกิดอะไรขึ้น?

✅ ผลดีที่คาดว่าจะเกิดขึ้น

  • แก้ systemic issue ได้ทุก listing พร้อมกัน (CAM fee, ชื่อนักพัฒนา, keyword)
  • เพิ่มโอกาสติดอันดับ Google ด้วย title/keyword ที่อิงพฤติกรรมค้นหาจริง
  • เพิ่มโอกาสถูกอ้างอิงโดย ChatGPT / Perplexity / Gemini ผ่าน FAQ + schema markup (AEO)
  • Content ได้ทั้งภาษาไทยและอังกฤษพร้อมกัน — ไม่ต้องจ้างแปลแยก
  • ทีม content ทำงานเร็วขึ้น — มี draft headline/description/FAQ พร้อมใช้ทันที
  • scale ได้ — เพิ่ม listing ใหม่กี่รายการก็รันซ้ำได้ทันที

⚠️ สิ่งที่ต้องระวัง / เตรียมการ

  • ผลลัพธ์ระดับ BORDERLINE/FAIL ต้องมีคนตรวจก่อนเผยแพร่
  • ข้อมูลทำเล/คู่แข่งบางส่วนมาจาก training data ของ AI ควรสุ่มตรวจความถูกต้อง
  • ค่าใช้จ่ายแปรผันตามจำนวน listing — ควรกำหนดงบประมาณ/รอบการรันที่ชัดเจน
  • ต้องดูแล API key ให้ปลอดภัย (ปัจจุบันเก็บใน .env)
  • ควรมี process อัปเดต schema เว็บไซต์ให้รองรับ FAQ/markup ที่ AI สร้าง
07 — ใครทำ และทำอย่างไร

กระบวนการพัฒนาโปรเจกต์นี้

โปรเจกต์นี้พัฒนาโดย Attaphon ร่วมกับ Claude (Anthropic AI) ผ่าน Cowork — ใช้เวลาออกแบบและสร้างต้นแบบทั้งหมดภายในเวลาไม่กี่ชั่วโมง

1. กำหนดปัญหาและเป้าหมาย

วิเคราะห์ listing ตัวอย่าง 20 รายการ เพื่อหา pattern ของปัญหาที่เกิดซ้ำ (CAM fee, ชื่อนักพัฒนา, SEO keyword, FAQ)

2. ออกแบบ Pipeline 11 Agent

กำหนดบทบาทและ system prompt ของแต่ละ agent ให้ครอบคลุมตั้งแต่ดึงข้อมูล วิจัย เขียน copy จนถึงให้คะแนน — เขียนเป็นไฟล์ instruction (CLAUDE.md)

3. สร้าง Automation Runner

เขียนสคริปต์ Python ที่อ่าน CSV, ดึงข้อมูลหน้า listing, เรียก Claude API ตาม pipeline และบันทึกผลเป็น JSON/CSV

4. สร้าง Virtual Office Dashboard

ออกแบบหน้าเว็บ pixel-art แสดง agent ทำงานแบบ real-time พร้อมหน้าแสดงผลลัพธ์แบบ bilingual ที่ทีม content เปิดใช้งานได้ทันที

5. ทดสอบกับ Listing จริง

รันระบบกับ listing จริง 3 รายการจากเว็บไซต์ ได้ผลลัพธ์ PASS ทั้งหมด พร้อมรายงานสำหรับการประชุมนี้
