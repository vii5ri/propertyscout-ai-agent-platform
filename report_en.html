An AI System That Audits and Rewrites Property Listings — Automatically, in Thai & English

The PropertyScout AI Agent Platform is a simulated "Virtual Office" where 11 AI agents work together — from pulling raw listing data to generating new bilingual content, fact-checking, and scoring quality — to boost Google search ranking (SEO) and increase the chance that AI assistants like ChatGPT, Perplexity, and Gemini surface the listing (AEO).

01 — Overview

Why does this system exist?

An audit of the first 20 listings on PropertyScout.co.th surfaced several recurring, systemic issues that hurt both SEO ranking and the credibility of each listing.

⚠️

Issues found (sample of 20 listings)

• 20/20 missing CAM fee, electricity & water pricing
• 13/20 missing developer name (weakens trust signal)
• 15/20 titles not using the keywords people actually search
• 8/20 had wrong BTS/MRT distances or station names
• 0/20 had FAQ schema for AI search engines

🎯

Goal of the system

• Automatically analyze every listing through 11 AI agents
• Fill missing data + verify accuracy
• Rewrite title, headline, description, FAQs — in Thai and English
• Score quality (PASS / BORDERLINE / FAIL) with priority actions
• Export to JSON + CSV, ready for the content team to use

02 — How it works

The 11-Agent Pipeline

Each listing passes through all 11 agents in sequence — like a virtual team where every member handles their part of the work, then hands off to the next.

1
KANYA INTAKE
Pulls data from the listing page — title, type, price, area, floor, bedrooms/bathrooms, photo count, video, and flags possible duplicate listings
2
PANSA ENRICH
Researches missing data — developer name, year built, unit count, distance to nearest BTS/MRT, malls, hospitals, schools, and project facilities
3
SUWAN AUDIT
Assesses initial data completeness, flagging listings with too few photos, no CAM fee, or no developer name
4
PRIYA RESEARCH
Compares price-per-sqm against the local market, checks competing listings nearby, and benchmarks typical CAM fees for this project type
5
NIRAN FACTCHECK
Verifies that BTS/MRT distances and station names, year built, floor counts, and price-per-sqm are realistic and not contradictory
6
TAWAN SEO
Optimizes title and keywords for Google ranking, in both Thai and English — using the actual terms buyers search, e.g. "condo for rent near BTS"
7
WICHAI COPYWRITER
Writes new headline, description, and meta description in TH/EN — fact-based, avoiding vague claims like "luxury" without proof
8
DAOW AEO
Builds schema markup and a bilingual FAQ set so ChatGPT, Perplexity, and Gemini can easily surface the listing in answers
9
PLOYCHIT OPTIMISE
Reconciles output from agents 6-8 — aligning TH/EN keywords, ensuring complete bilingual FAQs, and ranking issues by priority
10
PHITCHAYA SCORE
Calculates the final score: completeness × 0.4 + accuracy × 0.3 + appeal × 0.3, and assigns a grade A–D
11
FIAT PUBLISH
Final decision — PASS publishes immediately / BORDERLINE needs priority fixes first / FAIL is sent back to intake with a checklist
03 — What it's built from

Technology & components

The entire system was designed and built using Claude (Cowork) as the primary tool, made up of four core parts.

🧠

AI Model — Claude (Anthropic)

The brain of the system. Used to analyze listing pages, research locations, write copy, and assign scores — handling Thai and English in a single pass.

claude-opus-4-8 (highest quality) claude-haiku-4-5 (fast / cheap — optional)
🐍

Automation Runner (Python)

The run_agents.py script reads a list of URLs from a CSV → fetches each listing page → sends it to Claude through the 11-agent pipeline → saves the results.

Python + Anthropic API
📊

Output files (JSON + CSV)

agent_results.json — the full multi-dimensional dataset for the dashboard · ps_summary.csv — score summary for management · ps_details.csv — ready-to-use bilingual copy for the content team

🖥️

Virtual Office Dashboard (Pixel UI)

A pixel-art web dashboard showing all 11 agents working in real time inside a simulated office, plus a "Listing Results" view to inspect scores, generated copy, and priority actions per listing.

HTML / React (.jsx)

06 — Future impact

What happens if this is rolled out?

✅ Expected upside

  • Fixes systemic issues across every listing at once (CAM fee, developer name, keywords)
  • Higher chance of ranking on Google with titles/keywords based on real search behavior
  • Higher chance of being cited by ChatGPT / Perplexity / Gemini via FAQs + schema markup (AEO)
  • Bilingual content generated together — no need for separate translation work
  • Faster content team workflow — ready-to-use draft headlines, descriptions, and FAQs
  • Scales easily — new listings can be re-run through the pipeline anytime

⚠️ What to watch / prepare for

  • BORDERLINE/FAIL results need human review before publishing
  • Some location/competitor data comes from the AI's training knowledge — spot-check for accuracy
  • Costs scale with listing volume — set a clear budget and run cadence
  • API key security needs care (currently stored in .env)
  • Website should be set up to support the FAQ/schema markup the AI generates
07 — Who built it, and how

Development process

This project was built by Attaphon together with Claude (Anthropic AI) via Cowork — the design and prototype were completed in a matter of hours.

1. Define the problem and goals

Audited 20 sample listings to find recurring patterns of issues (CAM fee, developer name, SEO keywords, FAQs)

2. Design the 11-agent pipeline

Defined each agent's role and system prompt — covering data extraction, research, copywriting, and scoring — written into a single instruction file (CLAUDE.md)

3. Build the automation runner

Wrote a Python script that reads a CSV, fetches each listing page, calls the Claude API through the pipeline, and saves results as JSON/CSV

4. Build the Virtual Office dashboard

Designed a pixel-art web UI showing agents working in real time, plus a bilingual results view the content team can use immediately

5. Pilot test on real listings

Ran the system against 3 real listings from the website — all returned PASS, producing the report used for this meeting
